# This test provided the following key takeaway points:
- Using the AzureActivity connector in MS Sentinel enabled me to detect potentially suspicious activity affecting my Azure cloud resources
- Sentinel analytics rules are powerful: if we have an enormous amount of events that would be impossible for a human eye to get around with, by using KQL queries we can easily filter out and display only those that we are interested in.
- On top of the analytics rules, we can create automation rules to also define what exactly do we want to do with the alerts and incidents: enrih them with playbooks, trigger a logic app or perform some other type of automated response in order to minimize the manual work for SOC analysts so they can focus on more complex problems.
